## 一、产品概述

### 1.1 产品定位

**一句话定位：** PawSmart 是一款用视觉 AI 帮助狗主人理解和训练狗的订阅制 iOS App。

**核心卖点：** "AI 看懂你的狗" — 拍照识别品种、AI 分析行为、生成个性化训练计划、AI 助手随时答疑。

**与竞品的本质区别：**


| 竞品       | 核心模式           | 我们的区别                                      |
| -------- | -------------- | ------------------------------------------ |
| Dogo     | 视频课程 + 真人反馈    | 我们用 **AI 视觉分析** 替代真人反馈，成本更低、响应更快           |
| PawChamp | AI 文字聊天 + 真人专家 | 我们有 **图像/视频识别能力**（品种识别 + 行为分析），PawChamp 没有 |
| Woofz    | 按品种个性化 + 健康追踪  | 我们的个性化基于 **AI 实时分析**，不只是注册时填问卷             |
| Pupford  | 大量免费内容 + AI 聊天 | 我们的 AI 能力更强（视觉 + 语言），不只是文字问答               |


### 1.2 目标用户

**核心用户画像：**


| 维度       | 描述                                 |

## 二、市场依据

### 2.1 市场规模

- 狗训练 App 市场规模（2026）：**$11.7 亿美元**
- 预计规模（2035）：**$36.6 亿美元**，CAGR **13.5%**
- iOS 端占 **41%** 市场份额
- 北美占全球 **36%** 份额

### 2.2 进入窗口

**AI 差异化窗口正在关闭，但仍有 12-18 个月：**

- 老牌头部 3 家（Dogo 10M+ 用户、Woofz、Puppr）**仍无 AI 功能**
- PawChamp（2025年1月上线）用 Claude 驱动 AI 助手，已增长到 50 万用户
- 但 **没有竞品拥有视觉 AI 能力**（品种识别 + 行为分析）——这是我们的差异化核心
- 42% 新上线训练 App 集成了 AI 功能，AI 正在成为标配

### 2.3 竞品核心短板（= 我们的机会）

1. **订阅体验差** — 几乎所有竞品都有订阅投诉（价格不透明、取消困难）
2. **免费→付费边界粗暴** — 大多数免费版几乎不可用
3. **没有视觉 AI** — 无人做品种识别→训练推荐、行为视频分析
4. **内容深度不足** — 高级行为问题指导缺失
5. **自助 App（~$10/月）与真人 1v1（$34/周）之间存在巨大价格空白**

**F1.2 宠物档案创建**

**步骤 1 — 添加狗狗照片（核心差异化入口）**

```
界面元素：
- 标题："来，让我认识一下你的狗狗"
- 大面积拍照/上传区域（圆形裁切预览）
- [拍照识别] 按钮（主 CTA）
- [从相册选择] 按钮（次 CTA）
- [跳过，手动输入] 文字链接

拍照识别流程：
1. 调用摄像头 → 用户拍照
2. 显示分析动画（1-3秒）："AI 正在识别..."
3. 返回识别结果：
   - 品种名称 + 置信度（如：金毛寻回犬 95%）
   - 品种特征卡片（体型/性格/训练难度/常见行为问题）
   - "看起来不对？手动修改" 链接
4. 用户确认 → 进入下一步

AI 识别失败兜底（原有 3 种）：
- 置信度 < 70%：显示 Top 3 候选品种让用户选择
- 完全无法识别：引导手动选择品种（搜索 + 列表）
- 混血犬：标注为"混血"，展示最接近的品种特征
```

**品种识别完整错误路径（工程评审补充 — C2）：**

> 品种识别是引导流程的核心差异化入口，如果在这里挂掉，用户直接流失。以下定义 5 种额外错误场景的处理方式：


| #   | 错误场景                                   | 判断条件                                             | 用户端表现                                 | 兜底操作                                              |
| --- | -------------------------------------- | ------------------------------------------------ | ------------------------------------- | ------------------------------------------------- |
| E1  | **拍的不是狗**（猫、人脸、食物、风景）                  | AI 返回 `is_dog: false` 或狗类置信度 < 30%               | 显示友好提示："这好像不是一只狗哦 🐱 请拍一张你家狗狗的清晰照片"   | 停留在拍照页，不进入品种结果页。提供"手动选择品种"入口                      |
| E2  | **照片质量太差**（模糊、过暗、遮挡严重）                 | AI 返回 `quality_score < 0.4`（模型预检或后端评估）           | 显示提示："照片不太清晰，识别可能不准确。建议在光线充足的环境重新拍一张" | 提供两个按钮：[重新拍照] [仍然识别]（允许继续但降低期望）                   |
| E3  | **网络超时回退云端后仍失败**                       | 端侧推理超时(>3s) → 云端请求超时(>5s) 或返回错误                  | 显示："识别暂时不可用，请稍后再试"                    | 提供 [重试] + [手动选择品种] 两个按钮。失败不消耗每日免费次数               |
| E4  | **端侧 AI 模型未就绪**（首次使用，50MB 模型下载中/初始化失败） | 模型文件不存在或校验失败                                     | 首次使用时显示进度条："正在准备 AI 识别引擎（约 30 秒）"     | 模型下载失败 → 自动回退云端识别。如云端也不可用 → 引导手动选择。App 后台静默重试模型下载 |
| E5  | **相机/相册权限被拒**                          | `AVCaptureDevice.authorizationStatus == .denied` | 显示说明页："需要相机权限才能识别品种" + 权限用途说明         | 提供 [去设置中开启] 按钮（深链到系统设置）+ [从相册选择] 备选 + [手动选择品种] 兜底 |


**技术需求：**

- 品种识别模型：复用公司现有 AI 图像识别能力
- 支持品种数量：≥ 200 个 AKC 注册品种 + 混血犬
- 识别准确率目标：Top-1 ≥ 85%，Top-3 ≥ 95%
- 识别延迟：≤ 3 秒（端侧优先，超时回退云端）

**今日训练任务逻辑：**

- AI 根据用户的「训练需求问卷 + 品种 + 年龄 + 历史训练数据」生成每日 2-4 个训练任务
- 每个任务 3-15 分钟，总时长不超过用户设定的每日训练时间
- 任务按优先级排序：行为纠正 > 基础训练 > 进阶技巧
- 完成任务后自动更新进度，触发成就动画

**打卡系统（Streak）：**

- 每天完成至少 1 个训练任务 = 打卡成功
- 连续打卡 7 天 / 30 天 / 100 天 → 解锁成就徽章
- 断卡后显示鼓励文案："没关系，今天重新开始！"
- 连续 3 天未打开 App → 触发 Push 提醒

**课程内容元数据模型（工程评审补充 — C3）：**

> 定义课程的层级结构、免费标记规则和每日任务映射关系。此模型直接影响后端数据库设计和前端展示逻辑。

```
层级结构：Course → Lesson → Step

Course（课程/分类）
├── id: string (唯一标识)
├── title: string ("基础训练")
├── category: enum (basic | puppy | behavior | advanced | care)
├── total_lessons: int
├── thumbnail_url: string
└── lessons: [Lesson]

Lesson（单节课）
├── id: string (唯一标识)
├── course_id: string (所属课程)
├── title: string ("坐下训练")
├── order: int (课程内排序)
├── is_free: bool (免费标记 — 由后端控制，非客户端硬编码)
├── duration_minutes: int (预估时长)
├── difficulty: enum (beginner | intermediate | advanced)
├── breed_relevance: [string] (适用品种，空=所有)
├── video_url: string? (HLS 视频地址)
├── repeat_days: int (建议训练天数，如"坐下"=7天)
└── steps: [Step]

Step（训练步骤）
├── id: string
├── lesson_id: string
├── order: int
├── instruction: string ("手握零食举过狗头上方")
├── tip: string? (AI 个性化提示模板)
└── illustration_url: string? (步骤示意图)
```

**免费标记规则：**

- `is_free` 字段由后端管理，客户端根据此字段 + 用户订阅状态决定展示（锁定 / 可播放）
- 免费课程规则：基础训练前 5 节 + 幼犬专区前 2 节 = 7 节（见 4.2 权限矩阵）
- 后端 API 返回课程列表时，未订阅用户的非免费课程仅返回 metadata（标题/封面/时长），不返回 video_url 和完整 steps

**每日任务与课程的映射关系：**

```
每日任务生成逻辑
  │
  ├── 输入：用户档案(品种+年龄+训练历史) + 训练计划 + 可用课程池
  │
  ├── 输出：2-4 个 DailyTask
  │         DailyTask {
  │           lesson_id: string    ← 关联到具体 Lesson
  │           day_in_plan: int     ← 在该 Lesson 训练计划中的第几天
  │           is_review: bool      ← 是否为复习（上次标记"困难"的课程）
  │           estimated_minutes: int
  │         }
  │
  └── 规则：
      ├── 免费用户：只从 is_free=true 的 Lesson 池中选取
      ├── 完成 Lesson 的所有 repeat_days → 标记该 Lesson 完成
      ├── 完成一个 Lesson ≠ 完成一个 DailyTask（一天可有多个 Task 对应不同 Lesson）
      └── 完成当日所有 DailyTask = 打卡成功
```

##### F4.1 AI 品种识别

**用户故事：**

> 作为用户，我想拍照识别我的狗（或别人的狗）的品种，了解品种特征和训练建议。

**功能流程：**

```
拍照/上传 → AI识别(1-3s) → 品种结果卡片
                              ├── 品种名称 + 照片
                              ├── 品种特征（体型/性格/运动量/训练难度）
                              ├── 该品种常见行为问题
                              ├── 推荐训练课程（3-5个）
                              └── [设为我的狗] / [再拍一张]
```

**免费使用限制：**

- 免费用户：3 次/天
- Premium 用户：无限

**技术需求：**

- 复用公司 AI 图像识别 SDK
- 支持 ≥ 200 个品种
- 识别耗时 ≤ 3 秒
- 需要相机和相册权限

##### F4.2 AI 训练助手（聊天）

**用户故事：**

> 作为用户，我的狗突然开始乱叫，我想马上问 AI 该怎么处理，不用等预约训练师。

**功能描述：**

```
┌─────────────────────────┐
│  AI 训练助手              │
│  基于 [品种] [年龄] 回答  │ ← 上下文感知
├─────────────────────────┤
│                          │
│  用户: 我的金毛 4个月，    │
│  总是咬我的手，怎么办？    │
│                          │
│  AI: 4个月的金毛正在换牙   │
│  期，咬手是正常行为。      │
│  建议你：                 │
│  1. 被咬时立刻说"啊！"    │
│     并收回手              │
│  2. 递给她合适的磨牙玩具   │
│  3. 咬玩具时给予表扬      │
│                          │
│  📎 推荐课程：            │
│  [停止咬人 - 幼犬篇]      │ ← 可点击跳转
│                          │
├─────────────────────────┤
│  [输入问题...]        [发送]│
│  🎤 语音输入               │
└─────────────────────────┘
```

**AI 助手设计原则：**

- 回答基于用户宠物档案（品种、年龄、训练历史）个性化
- 使用正向强化训练法（Positive Reinforcement），不推荐惩罚方法
- 回答附带推荐课程链接
- 严重行为问题（攻击性）必须提示"建议咨询专业训练师"
- 支持文字和语音输入

**免费使用限制：**

- 免费用户：5 条/天
- Premium 用户：无限

**技术需求：**

- 使用公司 LLM 接口 或接入第三方 LLM API
- System Prompt 包含：品种知识库 + 训练方法论 + 安全边界
- 响应延迟：首 token ≤ 2 秒，完整回答 ≤ 8 秒（流式输出，见 7.3）
- 支持上下文对话（保留最近 20 轮）

##### F4.3 AI 训练计划生成器

**用户故事：**

> 作为用户，我不知道该按什么顺序训练，希望 AI 能根据我的狗的情况自动生成一个训练计划。

**功能描述：**

- 基于用户的「品种 + 年龄 + 行为问题 + 每日可用时间」自动生成 4-8 周训练计划
- 计划包含每周训练主题、每日任务安排、里程碑
- 可手动调整计划（跳过/替换某个课程）
- 训练进度影响计划动态调整（某课程标记"困难" → AI 增加复习次数）

**仅 Premium 用户可用。**

### 7.5 账号体系（工程评审补充 — A3）

> **背景：** 原 PRD 缺失注册/登录流程。订阅制 App 需要账号体系支撑用量计数、订阅绑定和跨设备同步。

#### 7.5.1 注册/登录流程

**支持的登录方式（按优先级）：**


| 方式                 | 优先级    | 说明                                     |
| ------------------ | ------ | -------------------------------------- |
| Sign in with Apple | P0（必须） | App Store 审核要求：提供第三方登录时必须同时提供 Apple 登录 |
| 邮箱+密码              | P0     | 兜底方案，支持所有用户                            |
| Google 登录          | P1     | 可选，提升转化率                               |


**登录时机：**

- **首次打开 App 不要求登录** — 用户先走完引导流程（拍照→问卷→付费墙）
- **以下场景触发登录：**
  - 用户选择开始免费试用 / 订阅（StoreKit 需要账号绑定）
  - 用户手动触发登录（设置页面）
- **匿名模式：** 引导流程中使用设备匿名 ID 暂存数据（宠物档案、问卷答案）；登录后自动合并到账号

**登录页面结构：**

```
┌─────────────────────────┐
│                          │
│  🐾 PawSmart             │
│                          │
│  [  Sign in with Apple ] │  ← 主 CTA（Apple 要求置顶）
│                          │
│  [  Sign in with Google] │  ← 次 CTA
│                          │
│  ─── 或使用邮箱 ───       │
│                          │
│  [邮箱地址.............. ]│
│  [密码.............. ]    │
│  [登录]                  │
│                          │
│  没有账号？注册            │
│  忘记密码？               │
│                          │
└─────────────────────────┘
```

**注册流程：**

1. 输入邮箱 + 密码（≥8位，含字母+数字）
2. 发送验证邮件（链接有效期 24h）
3. 验证成功 → 自动登录 → 合并匿名数据

**安全要求：**

- 密码哈希存储（bcrypt，cost ≥ 12）
- 登录失败 5 次 → 锁定 15 分钟 + 邮件通知
- JWT Token 有效期 30 天，支持 Refresh Token
- 登出时清除本地 Token，不清除缓存数据

#### 7.5.2 免费用量计数方案

**计数架构：服务端为准，客户端缓存**

```
用户操作 → 客户端检查本地缓存
  │
  ├── 有余量 → 允许操作 → 异步上报服务端
  │                          │
  │                          └── 服务端确认 → 更新本地缓存
  │
  └── 余量=0 → 请求服务端确认
                  │
                  ├── 确认达限 → 展示升级付费墙
                  └── 有余量（缓存过期）→ 允许操作 + 刷新缓存
```

**计数规则：**


| 维度     | 规则                                                         |
| ------ | ---------------------------------------------------------- |
| 「天」的定义 | **UTC 0 点**重置（而非本地时间，避免时区切换刷量）                             |
| 计数存储   | 服务端 Redis（key: `user:{id}:usage:{feature}:{date}`，TTL 48h） |
| 客户端缓存  | 本地 UserDefaults 缓存当日余量，每次操作后递减                             |
| 缓存刷新   | App 启动时 + 每次操作回调时从服务端同步                                    |
| 离线操作   | 允许本地计数，联网后与服务端对账（取 max）                                    |


**余量提示 UX：**


| 剩余次数  | 提示方式                                       |
| ----- | ------------------------------------------ |
| ≥ 2 次 | 不提示                                        |
| = 1 次 | 入口处显示 badge："最后 1 次"                       |
| = 0 次 | 触发软付费墙，文案："今天的 [功能] 次数已用完，升级 Premium 无限使用" |

### 7.6 离线与缓存优先策略（工程评审补充 — A1）

> **背景：** 宠物训练的典型场景是户外（公园、后院），网络不稳定。App 核心体验不能依赖实时网络。

#### 7.6.1 缓存策略矩阵


| 功能模块    | 离线策略    | 缓存时机                 | 缓存容量    |
| ------- | ------- | -------------------- | ------- |
| 每日训练任务  | ✅ 完全可用  | 每日任务生成时预加载 + 本地持久化   | ≤ 1MB   |
| 打卡/完成标记 | ✅ 离线排队  | 操作立即本地存储，联网后批量同步     | ≤ 100KB |
| 课程文字步骤  | ✅ 完全可用  | 用户浏览过的课程自动缓存         | ≤ 10MB  |
| 课程视频    | ❌ 需联网   | 不缓存（太大，且 CDN 有自适应码率） | —       |
| AI 训练助手 | ❌ 需联网   | 历史对话本地缓存（只读）         | ≤ 5MB   |
| AI 品种识别 | ⚠️ 端侧可用 | 端侧模型可离线推理；详细结果需联网    | 50MB 模型 |
| 宠物档案    | ✅ 完全可用  | 本地持久化 + 云端同步         | ≤ 2MB   |


#### 7.6.2 离线打卡同步机制

```
用户完成训练 → 立即写入本地 SQLite
  │
  ├── 网络可用 → 同步到服务端 → 标记已同步
  │
  └── 网络不可用 → 写入同步队列
                     │
                     └── 网络恢复（监听 NWPathMonitor）
                         │
                         └── 批量上传队列 → 服务端返回确认
                             │
                             ├── 成功 → 清除队列
                             └── 冲突 → 以服务端为准（last-write-wins）
```

**冲突处理规则：**

- 打卡数据：以「已完成」为准（不可回退已完成的训练）
- 训练计划：以服务端最新版为准
- 宠物档案：以时间戳最新的为准

#### 7.6.3 离线状态 UX


| 状态    | 用户可见表现                            |
| ----- | --------------------------------- |
| 完全离线  | 顶部显示淡色横幅："离线模式 — 你的训练进度会在联网后自动同步" |
| 需联网功能 | 功能入口正常可见，点击后弹出 toast："此功能需要网络连接"  |
| 同步中   | 无可见变化（静默同步）                       |
| 同步冲突  | 无打扰提示（自动解决）                       |

### 7.7 AI 安全边界方案（工程评审补充 — A2）

> **背景：** PRD 原有安全原则过于笼统。AI 聊天面临提示词注入、医疗越界、幼童安全等 4 类风险，需具体定义处理方式。

#### 7.7.1 四类禁区定义


| 类别          | 描述                     | 检测方式                                          | 处理方式                                                              |
| ----------- | ---------------------- | --------------------------------------------- | ----------------------------------------------------------------- |
| **惩罚性训练**   | 电击项圈、打骂、alpha roll 等   | System Prompt 硬编码 + 关键词过滤                     | 拒绝回答 + 固定回复："PawSmart 仅推荐正向强化训练法。惩罚性方法可能伤害你的狗并破坏信任关系。" + 推荐正向训练课程 |
| **医疗/健康建议** | 用药、诊断、症状判断             | 关键词检测（vomit, blood, limp, medicine, dosage 等） | 拒绝诊断 + 固定回复："这听起来可能是健康问题，建议尽快咨询兽医。我可以帮你准备就诊时的问题清单。"               |
| **不当内容**    | 虐待动物、繁殖建议、非法场景         | 内容审核层（moderation API）                         | 拒绝回答 + 记录日志 + 三次触发后限制对话                                           |
| **提示词注入**   | 尝试绕过 System Prompt 的指令 | 输入预处理：过滤 "ignore previous instructions" 等模式   | 静默忽略注入指令 + 正常回答用户本意（如能解析） + 连续 3 次注入尝试后返回固定回复                     |


#### 7.7.2 System Prompt 安全框架

```
SYSTEM PROMPT 结构：
├── 角色定义："你是 PawSmart AI 训练助手，专注于狗的正向强化训练指导。"
├── 行为边界：
│   ├── "始终推荐正向强化法（奖励正确行为，忽略错误行为）"
│   ├── "永远不推荐：电击项圈、打骗、alpha roll、呛水、夹嘴"
│   ├── "不提供兽医医疗建议，任何健康相关问题引导就医"
│   ├── "攻击性行为必须建议咨询认证训练师（CPDT-KA）"
│   └── "幼犬（< 6 月龄）的训练建议需特别标注注意事项"
├── 个性化上下文：
│   └── "用户的狗：{breed}, {age}, {name}, {training_history}"
└── 输出格式：
    └── "回答控制在 200 字以内，使用编号步骤，附带推荐课程"
```

#### 7.7.3 攻击性行为升级处理

用户描述攻击性行为

  ├── 轻度（对食物护食、低吼）

| 初次购买/试用开始 | 客户端 StoreKit                                                  | 客户端验证 receipt → 发送到服务端 → 服务端通过 App Store Server API 验证 → 更新 DB |
| 续费成功      | Server Notification V2 (`DID_RENEW`)                          | 服务端接收 → 验证 JWS → 更新 DB → 推送静默通知给客户端刷新                          |
| 取消（到期不续费） | Server Notification V2 (`DID_CHANGE_RENEWAL_STATUS`)          | 标记 `auto_renew_off` → 到期日前功能不变 → 到期后降级为 FREE                   |
| 退款        | Server Notification V2 (`REFUND`)                             | 立即降级为 FREE → 记录退款原因 → 触发挽留流程（可选）                               |
| 宽限期       | Server Notification V2 (`DID_FAIL_TO_RENEW` + `GRACE_PERIOD`) | 保持 ACTIVE 功能 → 最多 16 天 → 期间每次打开 App 提示更新支付方式                   |
| 恢复购买      | 客户端 `Transaction.currentEntitlements`                         | 遍历所有有效交易 → 匹配到有效订阅 → 恢复 ACTIVE                                 |

### 7.10 LLM 运行成本估算与控制策略（工程评审补充 — A4）

> **背景：** 定价年 $59.99（≈$5/月），Apple 30% 抽成后净收入 ≈$3.50/月。AI 功能成本必须控制在合理范围内。

#### 7.10.1 成本估算模型

**假设：** 使用 GPT-4o-mini 级别模型（$0.15/1M input tokens, $0.60/1M output tokens）


| AI 功能    | 每次调用 token 估算              | 免费用户（日均） | Premium 用户（日均） | 月成本/Premium 用户 |
| -------- | -------------------------- | -------- | -------------- | -------------- |
| AI 训练助手  | 入 800 + 出 400 tokens       | 2 条      | 8 条            | $0.14          |
| 训练计划生成   | 入 1200 + 出 600 tokens      | 0        | 0.3 次          | $0.05          |
| 品种识别云端兜底 | 入 2000 + 出 200 tokens（含图片） | 0.5 次    | 0.3 次          | $0.01          |
| **合计**   |                            |          |                | **$0.20（乐观）**  |


**场景分析：**

```
成本敏感度分析
═══════════════════════════════════════════════

场景          每用户月AI成本    净收入占比    风险等级
────          ────────────    ──────────    ────────
乐观（低活跃）   $0.20          5.7%        ✅ 安全
基准（正常活跃） $0.50          14.3%       ✅ 安全
悲观（高活跃）   $1.20          34.3%       ⚠️ 需关注
极端（滥用）     $2.50+         71%+        ❌ 亏损

安全线：AI 成本/净收入 ≤ 25%（$0.88/用户/月）
告警线：AI 成本/净收入 > 40%（$1.40/用户/月）
```

#### 7.10.2 成本控制策略


| 层级    | 策略              | 触发条件     | 实现方式                                          |
| ----- | --------------- | -------- | --------------------------------------------- |
| L1 基础 | 模型选型优化          | 默认启用     | 对话用 GPT-4o-mini；计划生成用模板 + LLM 个性化（减少纯 LLM 生成） |
| L2 缓存 | 常见问题缓存          | 默认启用     | 品种百科、常见训练问题用语义缓存（embedding 相似度 > 0.95 直接返回缓存） |
| L3 限流 | Premium「合理使用」限制 | 超出合理范围时  | AI 助手 50 条/天上限、计划生成 5 次/天上限（远高于正常使用量，仅防滥用）    |
| L4 降级 | 高峰时段模型降级        | 全局成本告警触发 | 高峰时段临时降级为更便宜模型（如 GPT-4o-mini → Haiku）；用户无感知   |
| L5 熔断 | AI 功能临时关闭       | 月成本预算耗尽  | 极端情况：关闭 AI 助手，保留品种识别（端侧无成本）；向用户显示"AI 助手维护中"   |


#### 7.10.3 LLM 选型建议


| 候选                      | 适用场景         | 成本                 | 优缺点             |
| ----------------------- | ------------ | ------------------ | --------------- |
| **GPT-4o-mini（推荐 MVP）** | AI 助手 + 计划生成 | $0.15/$0.60 per 1M | 成本低、质量可接受、流式支持好 |
| Claude Haiku            | 备选/降级方案      | $0.25/$1.25 per 1M | 安全性更好、成本略高      |
| 公司自研 LLM（如有）            | 长期方案         | 需确认                | 无 API 调用成本，但需运维 |


**架构要求：** LLM 调用层必须设计为可切换的抽象接口（`protocol LLMProvider`），后端可通过配置切换模型而无需发版。
